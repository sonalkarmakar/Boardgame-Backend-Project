# Board Game Backend
## Overview
This project is for demonstrating a typical CI/CD pipeline of an application, starting from the source code, ending with monitoring the deployed application, using a sample application about board games.

The source code of the application is acquired from [Boardgame](https://github.com/jaiswaladi246/Boardgame) project by [Aditya Jaiswal](https://github.com/jaiswaladi246). See that project's page for more details about the application.

## Tech Stack
The project uses the following tools:
- **Infrastructure**
	- [_AWS_](https://aws.amazon.com/what-is-aws/): hosts the infrastructure resources.
	- [_Terraform_](https://developer.hashicorp.com/terraform/intro): automates provisioning of infrastructure and other resources.
	- [_Ansible_](https://www.redhat.com/en/ansible-collaborative): prepares the host environment for deployment of other tools.
	- [_Linux_](https://www.linux.com/what-is-linux/): OS used for hosting the CI/CD tools.
- **Source code operations**
	- [_Git_](https://git-scm.com/book/en/v2/Getting-Started-What-is-Git%3F): version control system for project source code.
	- [_GitHub_](https://github.com/why-github): remote Git repository for project source code.
	- [_Maven_](https://maven.apache.org/what-is-maven.html): building and packaging application source code.
	- [_SonarQube_](https://docs.sonarsource.com/what-is-sonarqube): scan generated builds for issues.
	- [_Nexus Repository_](https://help.sonatype.com/en/product-information.html#:~:text=Nexus%20Repository%20is%20a%20universal%20repository%20manager%20that%20serves%20as%20a%20central%20hub%20for%20storing%2C%20managing%2C%20and%20distributing%20software%20artifacts%2C%20such%20as%20libraries%2C%20dependencies%2C%20and%20build%20outputs%2E): store different build versions generated by Maven.
	- [_Jenkins_](https://www.jenkins.io/doc/#what-is-jenkins): automating Maven and other build-related operations.
	- [_Trivy_](https://trivy.dev/): scan builds and images created from source-code for vulnerabilities.
- **Deployment**
	- [_Docker_](https://docs.docker.com/get-started/docker-overview/): deploy containers for CI/CD tools and create images for application builds.
	- [_Kubernetes_](https://kubernetes.io/docs/concepts/overview/#:~:text=Kubernetes%20is%20a%20portable%2C%20extensible%2C%20open%20source%20platform%20for%20managing%20containerized%20workloads%20and%20services%20that%20facilitate%20both%20declarative%20configuration%20and%20automation%2E): deploy application in cluster.
	- [_AWS Elastic Kubernetes Service_](https://docs.aws.amazon.com/eks/latest/userguide/what-is-eks.html): deploy application cluster hosted in AWS.
- **Monitoring**
	- [_Blackbox Exporter_](https://github.com/prometheus/blackbox_exporter#blackbox-exporter:~:text=The%20blackbox%20exporter%20allows%20blackbox%20probing%20of%20endpoints%20over%20HTTP%2C%20HTTPS%2C%20DNS%2C%20TCP%2C%20ICMP%20and%20gRPC): probe application endpoint (website) to check response and functionality.
	- [_Grafana_](https://grafana.com/docs/grafana/latest/#overview): plot application metrics in dashboard for visualisation.
	- [_Prometheus_](https://prometheus.io/docs/introduction/overview/#what-is-prometheus): record metrics about application endpoint (website) availability from Blackbox Exporter.

## Notable Features
- Mostly **automated** processes through Terraform and Ansible, requiring manual effort only for credentials, and initial setup of Jenkins, Nexus, SonarQube and Grafana.
- **Modular structure** for customisability through variables.
- **Generates and deletes new credentials locally** for every infrastructure deployment and destruction.
- **Containerised tools** for fast and easy configuration through replicable environment.
