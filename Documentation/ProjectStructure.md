# Project Structure
## Overview
This document contains the details about the different components required for this project.

The project uses the following tools:
- **Infrastructure**
	- _AWS_: hosts the infrastructure resources.
	- _Terraform_: automates provisioning of infrastructure and other resources.
	- _Ansible_: prepares the host environment for deployment of other tools.
	- _Linux_: OS used for hosting the CI/CD tools.
- **Source code operations**
	- _Git_: version control system for project source code.
	- _GitHub and GitLab_: remote Git repository for project source code.
	- _Maven_: building and packaging application source code.
	- _SonarQube_: scan generated builds for issues.
	- _Nexus Repository_: store different build versions generated by Maven.
	- _Jenkins_: automating Maven and other build-related operations.
	- _Trivy_: scan builds and images created from source-code for vulnerabilities.
- **Deployment**
	- _Docker_: deploy containers for CI/CD tools and create images for application builds.
	- _Kubernetes_: deploy application in cluster.
	- _AWS Elastic Kubernetes Service_: deploy application cluster hosted in AWS.
- **Monitoring**
	- _Blackbox Exporter_: probe application endpoint (website) to check response and functionality.
	- _Grafana_: plot application metrics in dashboard for visualisation.
	- _Prometheus_: record metrics about application endpoint (website) availability from Blackbox Exporter.

## Infrastructure
### Amazon Web Services (AWS)
Amazon Web Services (AWS) is used to host the following infrastructure resources required for the this project's demonstration:
- **Elastic Compute Cloud (EC2) Instances**: 
	- _Control Node_ instance: runs Ansible playbooks and creates the EKS cluster with cluster tools.
	- _Jenkins_ instance: runs Jenkins inside Docker container.
	- _Nexus Repository_ instance: runs Nexus Repository inside Docker container.
	- _SonarQube_ instance: runs SonarQube inside Docker containers.
	- _Monitoring_ instance: runs Blackbox Exporter, Grafana and Prometheus inside separate Docker containers in the same host.
	- _Cluster nodes_: automatically created by EKS configuration.
- **Security Group**: a separate Security Group is created inside the default VPC of the specified AWS Region. The SG must allow all outbound traffic and the following inbound traffic--
	- _SSH_: allow SSH traffic for manual configuration and Ansible plays.
	- _Kubernetes_: allow `kubectl` to connect with the EKS cluster.
	- _Nexus Repository_: allow storing and retrieving builds in the repository.
	- _Jenkins_: allow opening the Jenkins portal for build automation.
	- _SonarQube_: allow opening SonarQube portal for configuration and send source code for scanning.
	- _Blackbox Exporter_: allow probing target application and receiving response from application.
	- _Grafana_: allow connecting to Grafana portal for dashboard configuration and data visualisation.
	- _Prometheus_: allow storing metrics about probing application.
- **Identity and Access Management (IAM)**: for better security, the tasks can be delegated to IAM users with just enough privileges for performing them.
	- _Root User_: a Root User is necessary to have access to AWS resources, however it should be avoided as much as possible because of its highest privileges.
	- _Infrastructure Admin_: an IAM user able to provision and delete AWS infrastructure resources.
	- _Cluster Admin_: an IAM user for administrating the EKS clusters.
- **Elastic Kubernetes Service**: hosts the application's Kubernetes cluster in spcified AWS Region.

### Terraform
Terraform is used to automate provisioning the AWS infrastructure resources, except EKS (more details below).

The Terraform code is separated into the following modules based on the target resources:
- **EC2**: Handles creation of EC2 instances with specified configuration. Requires SSH key pair for remote access, and Security Group for external traffic.
	- _Variables_:
		- `instance_name`: name of the instance.
		- `instance_type`: type of AWS EC2 instance.
		- `root_vol_size`: size of root volume of the instance.
		- `ssh_public_key`: public SSH key used for connecting to instance.
		- `instance_sg`: AWS Security Group that the instance is connected to.
		- `user_data`: script to execute when creating the instance.
	- _Outputs_:
		- `public_ip`: public IP address of the instance.
- **IAM**: Creates an IAM user with the [minimum policies requried](https://docs.aws.amazon.com/eks/latest/eksctl/minimum-iam-policies.html) for EKS cluster administration, and an AWS Access Key.
	- _Variables_:
		- `project_prefix`: prefix used for naming AWS resources of the project.
		- `username`: unique username of the IAM user.
		- `user_path`: AWS path where the IAM user is created.
		- `display_name`: name shown in AWS Console.
		- `user_purpose`: purpose of the IAM user as shown in AWS Console.
		- `managed_policies`: preset IAM policies provided by AWS.
		- `inline_policies`: set of JSON files defining custom policies for the user.
	- _Outputs_:
		- `eks_admin_key_id`: AWS Access Key ID generated by the module.
		- `eks_admin_key_secret`: AWS Access Key secret generated by the module.
- **SecurityGroup**: Creates a Security Group inside the default VPC of the specified AWS region to connect the EC2 instances for external traffic.
	- _Varilables_:
		- `infra_region`: AWS region where the infrastructure will be created.
		- `project_prefix`: prefix used for naming AWS resources of the project.
		- `sg_name`: name of the security group.
		- `inbound_access_port`: list of ports required by the project to receive external traffic.
- **Main**: Main module takes input from the user for creating the different resources in the required sequence.
	- _Providers_:
		- `aws`: for provisioning AWS resources.
		- `tls`: for creating SSH key pair locally.
		- `local`: for creating and reading files locally.

### Ansible
Ansible is used to automate preparing the EC2 instances for running the tools required. The playbooks use the inventory named `inventory.ini` for managed nodes and certain variables.

The playbooks are listed below:
- `01_InstallDocker.yaml`: installs Docker, starts the `systemd` service and adds the Ansible user to the Docker group.
- `02_RunDockerContainers.yaml`: runs the Docker containers for Jenkins, Nexus Repository and SonarQube.
- `03_ConfigureJenkinsContainer.yaml`: installs plugins and other additional tools in the Jenkins container.
- `04_InstallClusterTools.yaml`: installs AWS CLI v2, Eksctl and `kubectl` for EKS cluster administration.
- `05_RetrieveInitialPasswords.yaml`: retrieves the initial admin passwords for Jenkins and Nexus Repository from their respective containers.
- `06_RunMonitoringContainers.yaml`: runs the Docker containers for Blackbox Exporter, Grafana and Prometheus.
